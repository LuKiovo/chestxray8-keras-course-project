# 基于 TensorFlow/Keras 的 ChestX-ray8 胸部 X 光多标签分类实验报告

## 摘要

本项目围绕 ChestX-ray8 胸部 X 光公开数据集，设计并实现了一个基于 TensorFlow/Keras 的十四类胸部疾病多标签分类系统。项目的核心目标不是构建真实临床诊断产品，而是在课程实训场景下完整覆盖深度学习项目的关键环节：数据清洗、标签编码、训练集划分、迁移学习建模、云 GPU 训练、测试集评估、结果可视化和交互式网页展示。ChestX-ray8 数据集中一张 X 光片可能同时对应多个疾病标签，因此本项目将任务建模为多标签分类问题，输出十四个独立疾病概率，并使用 sigmoid 激活与二元交叉熵类损失进行训练。

由于个人本机没有可用于深度学习训练的独立 GPU，本项目采用 RTX 4090 云 GPU 完成主要训练过程。考虑到云 GPU 数据盘容量有限，项目没有一次性复制全量训练图像进入训练目录，而是将训练集切分为多个 shard，按分片依次训练并保存 checkpoint。最终实验使用 111117 张有效图像，其中训练集 77782 张、验证集 11115 张、测试集 22220 张，训练集被划分为 16 个分片。模型采用 MobileNetV2 作为主干网络，并加载 ImageNet 预训练权重进行迁移学习。测试集评估结果显示，模型在 14 类疾病上的 Macro AUC 达到 0.7254，Micro F1 为 0.1681，Macro F1 为 0.1497。受类别不均衡、数据源中少量损坏图片以及阈值未针对每类单独优化等因素影响，F1 指标仍有提升空间，但 AUC 结果表明模型已经具备一定的疾病排序和区分能力。

项目还实现了 Streamlit 交互式网页，支持查看总体指标、逐疾病指标、训练曲线、ROC 曲线和文件产物状态，并保留单张 X 光图像预测入口。最终项目文件不包含原始数据集和大模型产物，便于按课程要求打包提交；训练结果、模型权重和评估数据作为实验产物另行保存。

## 1. 课题背景与任务分析

胸部 X 光片是临床医学中使用频率很高的影像检查方式之一。它具有检查成本较低、成像速度快、适用范围广等特点，常用于肺部感染、气胸、胸腔积液、心脏扩大、结节等疾病的初步筛查。随着深度学习技术的发展，卷积神经网络在医学影像识别任务中表现出较强的特征学习能力。和传统图像处理方法相比，深度模型能够从大量样本中自动学习边缘、纹理、局部结构以及更高层次的语义特征，减少人工特征设计的工作量。

本课程项目选题聚焦 ChestX-ray8 数据集。该数据集包含大规模胸部 X 光图像及对应疾病标签，常被用于胸部疾病自动识别相关研究。与普通单标签分类不同，ChestX-ray8 的一张图片可能同时存在多种疾病征象，例如同时出现 Atelectasis 和 Effusion。因此，本项目不能使用 softmax 将各疾病视为互斥类别，而应采用多标签学习方式，让模型为每个标签分别输出一个概率。模型输出维度为 14，对应 Atelectasis、Cardiomegaly、Effusion、Infiltration、Mass、Nodule、Pneumonia、Pneumothorax、Consolidation、Edema、Emphysema、Fibrosis、Pleural_Thickening 和 Hernia。

从产品和工程角度看，本项目并不是简单地写一个训练脚本，而是要形成一个完整、可复现、可展示的深度学习课程项目。它需要解决以下问题：第一，如何把原始 CSV 标签转成模型可用的多热向量；第二，如何在数据源不完全规范、存在少量坏图的情况下完成数据清洗；第三，如何在云 GPU 容量有限的环境里完成全训练集的分阶段训练；第四，如何使用固定验证集和测试集评估模型，避免每个阶段的结果不可比较；第五，如何把训练结果以图表和网页形式展示，方便答辩和课程验收。

## 2. 数据集与预处理

本项目使用 ChestX-ray8/ChestXray14 体系数据。由于下载源为拆分转载版本，实际解压后存在少量缺失或损坏图片。项目首先根据 `Data_Entry_2017.csv` 中的 `Image Index` 字段扫描本地图片目录，只保留真实存在的图像记录。之后训练过程中又发现个别 PNG 文件虽然存在，但内容损坏，TensorFlow 解码时报 `Invalid PNG data`。针对这种情况，项目对对应 shard 的 manifest 进行二次过滤，将损坏图像从训练清单中移除，并在 `bad_images.txt` 中记录。最终纳入实验的有效图片数为 111117 张。

数据集划分采用固定训练集、验证集和测试集。最终划分结果如下：训练集 77782 张，验证集 11115 张，测试集 22220 张。训练集进一步被划分成 16 个 shard，用于适配云 GPU 数据盘限制。每个 shard 大约包含 5000 张训练图片，最后一个 shard 因总数不能整除而包含 2782 张图片。验证集和测试集保持固定，不参与分片轮换。这样做的好处是每个阶段训练后都可以在同一验证集上观察模型变化，最终测试指标也具有一致性。

标签方面，项目从 `Finding Labels` 字段中解析疾病名称。如果一张图片包含多个疾病标签，则将这些标签对应位置置为 1，其余位置置为 0。如果图片标注为 `No Finding`，则 14 个疾病标签均为 0。根据清单统计，各标签样本数量差异明显。例如 Infiltration 有 19720 条阳性记录，Effusion 有 13236 条，Atelectasis 有 11464 条，而 Hernia 只有 223 条，Pneumonia 也只有 1419 条。这种长尾分布是医学影像多标签分类中的典型问题，会导致模型更容易学习高频疾病，而低频类别的召回和 F1 往往偏低。

图像输入统一缩放到 224 x 224，并归一化到 0 到 1 范围。选择 224 的原因是它与 MobileNetV2 的常见输入尺寸一致，可以直接利用 ImageNet 预训练权重。在训练阶段，数据通过 `tf.data.Dataset` 管道读取，批量大小设置为 32。由于项目重点是课程实训和整体流程复现，没有在第一版中过度引入复杂数据增强；后续如果继续优化，可以加入随机翻转、轻微旋转、对比度变化等医学影像相对安全的数据增强策略。

## 3. 模型设计与训练方法

项目选择 MobileNetV2 作为基础网络。MobileNetV2 的优势是参数量较小、推理速度较快、对显存和计算资源较友好，适合课程项目和云 GPU 成本受限的场景。与 ResNet、DenseNet 等更重的模型相比，MobileNetV2 的性能上限可能略低，但它能够较快跑完整个训练、评估和展示流程。在课程评分更关注深度学习完整实践的前提下，使用轻量模型进行迁移学习是比较稳妥的方案。

模型结构上，主干网络负责提取图像特征，顶部接全局池化和全连接分类层，最终输出 14 个 sigmoid 概率。由于该任务是多标签分类，项目没有使用 softmax，也没有强制各类别概率和为 1。每个疾病标签可以被独立判断，因此模型可以同时预测多个阳性疾病。训练损失使用二元交叉熵类损失，并保留类别权重设计以缓解阳性样本稀少带来的不平衡问题。

训练采用分阶段方式完成。首先使用 `prepare_data.py` 生成全量 manifest、训练集、验证集、测试集以及 shard 索引；随后从 `train_shard_000.csv` 开始训练，保存 `best_model.keras`、`last_model.keras`、`training_log.csv` 和 `training_summary.json`。训练下一个 shard 时加载上一个 shard 的 `best_model.keras` 作为起点继续训练。这样做相当于让模型按顺序看完整个训练集，既避免一次性复制大量图片，又保留了连续训练的效果。最终模型训练至 `shard_015`，即完成全部 16 个训练分片。

云 GPU 环境为 RTX 4090，显存 24GB。训练过程中每个 shard 设置 5 个 epoch，batch size 为 32，学习率为 0.0001。由于最后一个 shard 只有 2782 张训练图像，最终训练摘要中显示最后阶段训练图片数为 2782，验证图片数为 11115。最后一个 shard 的最终训练 AUC 为 0.7318，验证 AUC 为 0.7308，训练 loss 为 1.1500，验证 loss 为 1.1556。从训练曲线看，训练指标与验证指标接近，没有出现非常明显的过拟合发散现象。

## 4. 分阶段训练工程实现

本项目的一个工程重点是适配云 GPU 的磁盘限制。ChestX-ray8 全量数据规模较大，如果同时保存压缩包、解压图片、训练中间产物和模型文件，很容易占满数据盘。因此项目设计了 manifest 和 staged training 的流程。原始图片保存在数据盘统一目录中，manifest 中记录每张图片的相对路径和标签信息。训练脚本只依赖 CSV 清单与图片根目录，不要求把数据复制进项目仓库。

最初设计中，`stage_shard.py` 可以把当前 shard 的图像复制到 staging 目录中，再从 staging 目录训练。实际云端运行时发现，如果训练集在 staging 目录，而验证集仍使用同一个 image root，就会导致验证集图片找不到。为保证训练稳定，本次实验改为使用 `all_images` 软链接目录作为统一图片根目录。该目录将分散在不同子目录中的 PNG 图片统一映射为扁平文件结构，既避免每次递归查找图片带来的性能损耗，也让训练集、验证集和测试集都能从同一个 image root 读取图片。

自动训练脚本 `overnight_train.sh` 用于无人值守训练。脚本会遍历 `manifests/shards/train_shard_*.csv`，检查对应 `outputs/shard_xxx/best_model.keras` 是否已经存在；如果存在则跳过，方便中断后续训。如果不存在，则加载前一个 shard 的最佳模型继续训练。训练完成全部 shard 后，脚本自动执行测试集评估，并生成 `metrics_summary.json`、`roc_curves.json` 和 `predictions.csv`。实验中 shard 012 和 shard 013 遇到损坏 PNG，项目通过过滤对应 CSV 后重新运行脚本，成功从中断位置继续训练。

这种流程体现了课程项目中的工程可复现性：训练不是一次性手工命令，而是可以断点续跑、保留日志、记录坏图并输出统一评估结果。最终项目没有把原始数据集和模型权重提交到 GitHub，而是通过 `.gitignore` 排除 `outputs/`、`manifests/`、`*.keras`、`*.tar.gz` 等大文件，只提交代码、配置、文档和可复现说明。

## 5. 实验结果与分析

最终测试集包含 22220 张图片。整体指标如下：Macro AUC 为 0.7254，Macro F1 为 0.1497，Micro F1 为 0.1681，Subset Accuracy 为 0.0919。对于多标签医学影像任务，Subset Accuracy 要求一张图片的所有标签完全预测正确，因此通常数值较低，不能单独作为模型优劣的主要依据。AUC 更适合衡量模型对阳性和阴性样本的排序能力，因此本报告重点分析 AUC。

从逐标签 AUC 看，模型在 Edema 上达到 0.8321，在 Effusion 上达到 0.7993，在 Emphysema 上达到 0.7675，在 Pneumothorax 上达到 0.7616，在 Hernia 上达到 0.7561，在 Cardiomegaly 上达到 0.7479。这说明模型对部分具有较明显影像特征的疾病具备较好的区分能力。Atelectasis 的 AUC 为 0.7177，Fibrosis 为 0.7225，Pleural_Thickening 为 0.7044，也达到了一定水平。相对较弱的类别包括 Nodule、Mass、Pneumonia 和 Infiltration，其中 Nodule AUC 为 0.6289，Mass AUC 为 0.6514，Pneumonia AUC 为 0.6551，Infiltration AUC 为 0.6662。

F1 指标整体偏低，主要原因有三点。第一，数据类别极度不平衡，低频疾病的阳性样本非常少。例如 Hernia 测试集中阳性样本只有 54 个，Pneumonia 只有 273 个，模型很难在固定阈值下取得稳定 F1。第二，本实验统一使用 0.5 作为所有标签阈值，但不同疾病的最佳阈值并不相同。如果在验证集上为每个标签单独寻找最优阈值，F1 有较大概率提升。第三，ChestX-ray8 标签来自文本挖掘和弱标注，标签噪声会影响模型学习。对于医学影像来说，单纯依赖图像级标签训练也较难精准定位病灶。

从训练过程看，最后一个 shard 的验证 AUC 约为 0.7308，与测试 Macro AUC 0.7254 接近，说明验证表现和测试表现较一致，没有出现明显的评估泄漏或严重过拟合。训练 loss 和验证 loss 也较接近。训练曲线、逐疾病柱状图和 ROC 曲线已通过可视化脚本生成，包括 `training_loss.png`、`training_auc.png`、`training_accuracy.png`、`label_auc.png`、`label_f1.png`、`label_accuracy.png` 和 `roc_curves.png`。这些图表可以用于报告插图和答辩展示。

## 6. 交互式网页展示

为了让结果更直观，项目使用 Streamlit 实现了交互式网页。网页包含项目概览、单图预测、评估指标、训练过程和文件产物几个标签页。项目概览页展示任务说明、标签数量、训练方式、模型框架和 14 类疾病标签。评估指标页读取 `metrics_summary.json`，展示 Macro AUC、Macro F1、Micro F1、Subset Accuracy 以及逐疾病 AUC、F1、Accuracy 和阳性样本数。训练过程页读取 `training_log.csv`，用表格和折线图展示 loss、AUC、binary accuracy 及验证指标。文件产物页检查模型、评估 JSON、ROC JSON、训练日志和图表目录是否存在，并展示已生成的 PNG 图表。

单图预测页保留上传图片接口。当本地具备 TensorFlow 环境并且模型文件可用时，用户可以上传 PNG、JPG 或 JPEG 格式的胸部 X 光图像，网页会调用训练好的 Keras 模型输出每类疾病概率，并根据阈值给出阳性标签。需要强调的是，该功能仅用于课程实验展示，不用于真实临床诊断。网页中展示的预测结果不能替代专业医生判断，也不能用于医疗决策。

为了适配本地结果包结构，网页默认会优先识别 `chestxray8_gpu_results/` 目录。如果该目录存在，模型、评估指标、训练日志和图表默认路径会自动指向云端下载结果；如果不存在，则回退到项目根目录下的 `outputs/`。这样既兼容云端运行，也方便本地展示。

## 7. 问题、改进与反思

本项目完成了从数据到网页的完整闭环，但仍有改进空间。首先，模型方面可以尝试 DenseNet121、EfficientNetV2 或 ConvNeXt 等更强主干网络。医学影像分类中 DenseNet 系列经常表现较好，因为密集连接有利于复用不同层级的纹理特征。其次，阈值策略可以优化。目前统一阈值 0.5 对所有疾病并不公平，后续可以在验证集上为每个标签搜索最大 F1 或 Youden 指数对应的阈值，再用于测试集评估。第三，可以加入更系统的数据增强，例如轻微旋转、平移、亮度对比度变化，但应避免破坏医学语义。

第四，可以改进类别不均衡处理。本项目已有类别权重设计，但还可以尝试 focal loss、重采样、类别平衡 batch 或对低频类进行更细致的损失加权。第五，可以在报告中增加混淆分析和错误样例可视化，观察模型在哪些疾病上更容易误判。第六，如果时间和算力允许，可以在训练后进行 fine-tuning，解冻主干网络后几层，以更小学习率继续训练，从而让 ImageNet 预训练特征更适应胸部 X 光灰度影像。

工程方面，本项目最有价值的经验是分阶段训练和断点续跑。实际云 GPU 训练过程中遇到了下载速度慢、数据盘路径混乱、坏图、图表脚本类型错误等问题。如果没有可恢复的训练脚本，每次错误都可能导致大量重复训练。通过按 shard 保存模型、检查已有 checkpoint、过滤坏图并继续训练，项目最终在有限时间内完成了全训练集训练和测试集评估。这种工程流程比单纯追求一次训练命令更符合真实机器学习项目的工作方式。

## 8. 结论

本项目基于 TensorFlow/Keras 完成了 ChestX-ray8 胸部 X 光 14 类多标签分类实验，实现了数据预处理、分阶段训练、测试集评估、图表生成和 Streamlit 网页展示。最终使用 111117 张有效图像，其中测试集 22220 张，完成 16 个训练分片的连续训练。模型在测试集上的 Macro AUC 为 0.7254，说明迁移学习模型已经具备一定的疾病区分能力。虽然 F1 指标受类别不均衡和统一阈值影响偏低，但项目已经形成了一个完整、可运行、可展示、可复现的深度学习课程项目。

从课程实训角度看，本项目体现了深度学习开源框架的完整应用：使用 TensorFlow/Keras 构建模型，使用 `tf.data` 读取图像，使用云 GPU 完成训练，使用 scikit-learn 计算 AUC、F1 和 ROC，使用 Matplotlib 输出图表，使用 Streamlit 构建交互网页。后续如果继续优化，可以从更强模型、阈值调优、类别不均衡处理和更丰富可解释性分析四个方向提升效果。

最后需要再次说明，本系统仅用于课程实验和辅助研究展示，不是临床诊断系统。任何医学判断都应由专业医生结合完整临床资料完成。
